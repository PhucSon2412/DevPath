# roadmaps_json — Data Structure Reference

> Generated by `crawl_roadmaps.py` + `crawl_node_content.py`. This is the schema reference for both scripts.

## Folder structure
```
roadmaps_json/
  _index.json          # index of all roadmaps
  frontend.json        # one file per roadmap
  backend.json
  ...
```

---

## `_index.json`
```json
{
  "total_roadmaps": 83,
  "roadmaps": [
    {
      "id": "frontend",
      "title": "Front-end",
      "type": "flow",          // "flow" | "content-only"
      "source_url": "https://roadmap.sh/frontend",
      "total_nodes": 137,
      "total_edges": 87,
      "node_types": ["title","topic","subtopic",...],
      "file": "frontend.json"
    }
  ]
}
```

## Roadmap file — type `"flow"` (59 roadmaps)

Full graph data with absolute pixel coordinates (ReactFlow format).

```json
{
  "id": "frontend",
  "title": "Front-end",
  "type": "flow",
  "source_url": "https://roadmap.sh/frontend",
  "stats": {
    "total_nodes": 137,
    "total_edges": 87,
    "nodes_with_content": 115   // added by crawl_node_content.py
  },
  "content_crawled_at": "ISO datetime",  // set after crawl_node_content.py runs
  "nodes": [ /* see Node schema */ ],
  "edges": [ /* see Edge schema */ ]
}
```

### Node schema
```json
{
  "id": "abc123",
  "type": "topic",
  "label": "HTML",
  "position": { "x": 100.5, "y": 200.0 },   // absolute pixel coords
  "size": { "width": 174, "height": 49 },
  "style": {
    "fontSize": 17,
    "justifyContent": "flex-start",
    "textAlign": "center",
    "backgroundColor": "#fff",    // optional
    "borderColor": "#000"         // optional
  },
  "z_index": 999,                 // sections use -999 (render behind)
  "resources": [                  // optional, legacy learning links
    { "title": "MDN HTML", "url": "https://...", "type": "article" }
  ],
  "priority": {                   // optional, visual hint
    "color": "#874efe",
    "label": "Personal Recommendation / Opinion"
  },
  "content": {                    // optional, added by crawl_node_content.py
    "description": "Short paragraph describing what this topic is.",
    "links": [
      {
        "title": "Link title",
        "url": "https://...",
        "type": "article"         // article|video|course|roadmap|feed|podcast|tool|book|official
      }
    ]
  }
}
```

### Node types — what to render
| type | render as | notes |
|------|-----------|-------|
| `title` | Large heading | Roadmap title |
| `topic` | Filled box, bold | Major skill group |
| `subtopic` | Outlined box | Specific skill/technology |
| `section` | Background rectangle | `z_index: -999`, no label, groups nodes visually |
| `paragraph` | Text box | Annotations / descriptions |
| `label` | Small caption text | Sub-heading within a section |
| `button` | Clickable link button | Links to other roadmaps |
| `linksgroup` | Group of related links | Related roadmaps panel |
| `vertical` | Thin vertical bar | Visual connector, no content |
| `horizontal` | Thin horizontal bar | Visual connector, no content |
| `legend` | Color legend box | Priority key |
| `checklist` | Checklist box | Task list |
| `todo` | Todo item | Learning task |
| `resourceButton` | Resource link | External resource |

> **Skip rendering**: `vertical`, `horizontal` — these are decorative connectors, not real nodes.  
> **Background layer**: `section` — render first (z-index behind all others).

### Edge schema
```json
{
  "id": "edge-abc",
  "source": "nodeId1",
  "target": "nodeId2",
  "source_handle": "x2",     // connection point: x1/x2=left/right, y1/y2=top/bottom
  "target_handle": "w1",
  "edge_style": "solid",     // "solid" = required skill | "dashed" = optional
  "line_type": "default",    // "default"(bezier) | "straight" | "step"
  "style": {
    "stroke": "#2b78e4",
    "stroke_width": 3.5
  }
}
```

**Edge meaning:**
- `solid` → Required / must-learn path
- `dashed` → Optional / alternative path

## Roadmap file — type `"content-only"` (24 roadmaps)

No graph. Only a list of markdown file URLs (no flow data).

```json
{
  "id": "claude-code",
  "title": "Claude Code",
  "type": "content-only",
  "stats": { "total_content_files": 112 },
  "content_files": [
    {
      "name": "introduction@RqfeqV.md",
      "path": "src/data/roadmaps/claude-code/content/introduction@RqfeqV.md",
      "url": "https://raw.githubusercontent.com/.../introduction%40RqfeqV.md"
    }
  ],
  "nodes": [],   // always empty
  "edges": []    // always empty
}
```

> Content file names follow pattern: `topic-slug@nodeId.md`. Fetch `url` to get markdown content.

## Key implementation notes

1. **Coordinate system**: `position.x/y` are absolute pixels on an infinite canvas. To render, translate so min(x,y) = 0, or use a canvas/SVG with pan+zoom.
2. **Rendering order**: Sort nodes by `z_index` ascending before rendering (sections at -999 go first as backgrounds).
3. **Which nodes matter**: For a learning roadmap UI, focus on `topic` and `subtopic` nodes — they contain the actual skill content. Skip `vertical`/`horizontal`/`section` for simplified views.
4. **Connection points** (`source_handle` / `target_handle`): Use first character to determine side — `x`=left/right, `y`=top/bottom, `w`/`z` = variants of left/right. Trailing `1` = start side, `2` = end side.
5. **Missing fields**: `resources`, `priority`, `style.*`, `content` are all optional — handle gracefully with defaults.
6. **node.content vs node.resources**: `content` (from `crawl_node_content.py`) is richer — has full description + categorized links. `resources` is a simpler legacy field from the raw flow JSON. Prefer `content` when available.
7. **Clickable nodes**: For a detail panel UI, nodes with `content` or `resources` can be clicked to show description + links. Main types: `topic`, `subtopic`.

## Scripts

| Script | Purpose |
|--------|---------|
| `crawl_roadmaps.py` | Crawl flow JSON + content-only markdown lists from GitHub. Run once to bootstrap. |
| `crawl_node_content.py` | Fetch `content/` folder for each flow roadmap, parse markdown, patch `content` field into nodes. Run after `crawl_roadmaps.py`. |
| `roadmap_viewer.html` | Web viewer: renders nodes/edges, sidebar, click-to-detail panel. Serve via `serve.py`. |
| `serve.py` | Simple Python HTTP server. Run `python serve.py` then open `http://localhost:8000`. |
